unit tests in the `web` and `mobile` artifacts. Do NOT add `vitest` to them.
**Why:** This repo is `node-linker=hoisted` (see `.npmrc`). Adding `vitest` to an
artifact pulls `vite` as a dependency, and once enough packages need it pnpm hoists a
**root-level** `node_modules/vite` copy. The `mockup-sandbox` artifact's `vite.config.ts`
uses the advanced builder/environment API (`builder.buildApp`, `environments`,
`hotUpdate`), which is strict about `Plugin` type identity — so a second physical vite
